⚡ COME USARE - MODO PIÙ SEMPLICE

# Una volta clonata la repository, basta:

    cd s4t_crossplane_deployment
    ./setup-all.sh

E aspetta 5-10 minuti. Fatto!

═══════════════════════════════════════════════════════════════════════════════

🔄 COSA FA AUTOMATICAMENTE setup-all.sh

PASSO 1:  Valida la struttura della repository
PASSO 2:  Installa k3s (se non presente)
PASSO 3:  Configura kubeconfig
PASSO 4:  Installa Helm (se non presente)
PASSO 5:  Verifica connessione cluster
PASSO 6:  Genera certificati TLS (Keycloak, Crossbar, CA)
PASSO 7:  Crea Keycloak ConfigMaps
PASSO 8:  Crea Keystone ConfigMaps
PASSO 9:  Esegue preflight checks
PASSO 10: Crea SSL certificate ConfigMaps
PASSO 11: Esegue script principale deployment
PASSO 12: Configura certificati SSL in PVC
PASSO 13: Verifica stato deployment
PASSO 14: Mostra info di accesso

═══════════════════════════════════════════════════════════════════════════════

🛠️ OPZIONI AVANZATE

Se hai già k3s o Helm installati:

    ./setup-all.sh --skip-k3s           # Salta k3s (usa cluster esistente)
    ./setup-all.sh --skip-helm          # Salta Helm (usa Helm esistente)
    ./setup-all.sh --skip-k3s --skip-helm  # Salta entrambi

═══════════════════════════════════════════════════════════════════════════════

🔍 VERIFICARE IL DEPLOYMENT

Dopo il setup, verifica che tutto è ok:

    ./verify-deployment.sh

Questo controlla:
  ✓ Connessione cluster Kubernetes
  ✓ Stato di tutti i pod
  ✓ Disponibilità servizi
  ✓ ConfigMaps creati
  ✓ Keycloak e Keystone status
  ✓ LoadBalancer IP assegnato
  ✓ Connettività API
  ✓ URL di accesso dashboard

═══════════════════════════════════════════════════════════════════════════════

🌐 ACCESSO DASHBOARD

Una volta completato il setup, accedi a:

URL: http://<LoadBalancer-IP>/horizon
     oppure http://<Node-IP>:31123/horizon

Username: (da `.env`: `STACK4THINGS_ADMIN_USER`)
Password: (da `.env`: `STACK4THINGS_ADMIN_PASSWORD`)

Lo script mostra automaticamente l'URL corretto al termine.

🎯 ALTERNATIVA: USANDO MAKE

Se preferisci usare Make anziché bash:

    make setup              # Esegue setup completo
    make setup-skip-k3s     # Con k3s esistente
    make status             # Mostra stato deployment
    make logs               # Segue log servizi
    make clean              # Rimuove deployment (mantiene k3s)
    make clean-all          # Disinstalla k3s completamente
    make help               # Mostra tutte le opzioni

🔧 COSA VIENE INSTALLATO

Infrastructure:
  - k3s (lightweight Kubernetes)
  - Helm (package manager)
  - MetalLB (load balancer)
  - Istio (service mesh)

Stack4Things:
  - IoTronic Conductor (gestione centrale)
  - Crossbar (router WAMP)
  - Wagent (agente IoT)
  - UI (Horizon dashboard)
  - RabbitMQ (message broker)
  - Database (MariaDB)

Identity:
  - Keycloak (OIDC provider)
  - Keystone (OpenStack federation)

Infrastructure-as-Code:
  - Crossplane (declarative management)
  - Custom S4T Provider

# Disinstalla tutto
